En cada proyecto necesitas proteger rutas. Algunas solo para usuarios autenticados, otras para administradores, otras solo para invitados. Escribes un `PrivateRoute`. Luego un hook para verificar roles. Luego un componente `HasAccess`. Pasas al siguiente proyecto — y lo escribes todo de nuevo.

La lógica siempre es la misma. La implementación siempre es diferente.

`react-protected` estandariza esto. Describes lo que cada ruta requiere. La librería verifica el acceso y redirige.

## Tres paquetes

`@react-protected/core` — lógica de control de acceso independiente del framework. Sin dependencias de React ni de ningún router.

`@react-protected/react` — contexto React (`AccessProvider`), hooks (`useHasAccess`) y componente `HasAccess`.

`@react-protected/react-router` — adaptador para React Router. Incluye todo de `@react-protected/react` — no necesitas instalar ambos.

## Instalación

Para proyectos con React Router:

```bash
npm install @react-protected/react-router
```

Para proyectos React sin React Router:

```bash
npm install @react-protected/core @react-protected/react
```

## Configuración basada en objetos (recomendado)

```ts
// router.ts
import { createAccessRouter } from '@react-protected/react-router'
import { useAuthStore } from './entities/auth'

export const router = createAccessRouter(
  [
    { path: '/', element: <HomePage /> },
    { path: '/login', element: <LoginPage />, access: 'guest-only' },
    { path: '/dashboard', element: <DashboardPage />, access: 'authenticated' },
    { path: '/admin', element: <AdminPage />, access: 'authenticated', roles: ['admin'] },
    {
      path: '/contracts',
      element: <ContractsPage />,
      access: 'authenticated',
      permissions: ['contracts:read'],
    },
    { path: '/403', element: <Page403 /> },
  ],
  {
    getUser: () => useAuthStore.getState().user,
    hasRole: (user, roles) => roles.some((role) => user.roles.includes(role)),
    hasPermission: (user, permissions) =>
      permissions.every((p) => user.permissions.includes(p)),
    loginPath: '/login',
    forbiddenPath: '/403',
    defaultPath: '/dashboard',
    callbackUrlParam: 'next',
  }
)

// App.tsx
import { RouterProvider } from 'react-router-dom'
export const App = () => <RouterProvider router={router} />
```

## Rutas JSX

```tsx
import { Route, Routes } from 'react-router-dom'
import { AccessProvider, AccessRoute } from '@react-protected/react-router'

const App = () => (
  <AccessProvider
    getUser={() => useAuthStore.getState().user}
    hasRole={(user, roles) => roles.some((role) => user.roles.includes(role))}
    loginPath="/login"
    forbiddenPath="/403"
    defaultPath="/dashboard"
    callbackUrlParam="next"
  >
    <Routes>
      <Route path="/" element={<HomePage />} />
      <Route
        path="/login"
        element={
          <AccessRoute access="guest-only">
            <LoginPage />
          </AccessRoute>
        }
      />
      <Route
        path="/dashboard"
        element={
          <AccessRoute access="authenticated">
            <DashboardPage />
          </AccessRoute>
        }
      />
    </Routes>
  </AccessProvider>
)
```

## Roles, permisos o ambos

RBAC — pasa `roles` a la configuración de la ruta. Tú defines la lógica de verificación: OR, AND, jerarquía — lo que tu dominio requiera.

ABAC — pasa `permissions`. Funciona igual.

Ambos a la vez — la verificación pasa solo cuando se cumplen ambas condiciones.

## Protección de elementos UI

Proteger rutas no es suficiente. `HasAccess` y `useHasAccess` permiten ocultar botones, campos o cualquier elemento UI basándose en rol o permiso — sin cambiar de ruta.

```tsx
import { HasAccess, useHasAccess } from '@react-protected/react-router'

// Componente
const Toolbar = () => (
  <nav>
    <HasAccess roles={['admin']}>
      <button>Eliminar usuario</button>
    </HasAccess>
  </nav>
)

// Hook
const ExportButton = () => {
  const canExport = useHasAccess({ permissions: ['reports:export'] })
  return canExport ? <button>Exportar</button> : null
}
```

## Callback URL

Cuando `callbackUrlParam` está configurado, los usuarios no autenticados son redirigidos conservando la ruta actual:

```
/dashboard?tab=overview → /login?next=%2Fdashboard%3Ftab%3Doverview
```

Maneja el retorno en tu página de login:

```ts
const [params] = useSearchParams()
navigate(params.get('next') ?? '/dashboard', { replace: true })
```

Si quieres suprimir el callback URL después de un cierre de sesión explícito:

```tsx
<AccessProvider
  callbackUrlParam="next"
  shouldAddCallbackUrl={() => !authStore.getState().loggedOut}
  ...
>
```

## ¿No usas React Router?

Usa `@react-protected/core` directamente con cualquier router. Ejemplo con TanStack Router:

```ts
import { createGuard } from '@react-protected/core'

const guard = createGuard({
  getUser: () => useAuthStore.getState().user,
  hasRole: (user, roles) => roles.some((role) => user.roles.includes(role)),
})

const dashboardRoute = createRoute({
  path: '/dashboard',
  beforeLoad: ({ location }) => {
    const result = guard.check({ access: 'authenticated' })
    if (!result.allowed) {
      throw redirect({
        to: result.reason === 'unauthenticated' ? '/login' : '/403',
        search: { next: location.pathname },
      })
    }
  },
  component: DashboardPage,
})
```
